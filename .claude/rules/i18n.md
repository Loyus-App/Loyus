# i18n Rules

## Setup

- react-i18next with synchronous init (`initImmediate: false`) -- no async loading flash
- Single default namespace (no multi-namespace setup)
- Types in `src/infra/i18n/types.ts`

## Locale Files

- Located in `src/infra/i18n/locales/`
- 6 languages: en, fr, es, pt, ru, de
- English is the source language; other translations must match key structure

## Key Conventions

- All keys use dot-notation: `t('section.key')`
- When adding a new user-facing string: add key to ALL 6 locale files
- Keys are organized by screen/feature section (e.g., `home.title`, `card.name`)

## Integration

- All user-facing strings in UI components wrapped in `t('key')` via react-i18next `useTranslation` hook
- Never use raw string literals for visible text in components
